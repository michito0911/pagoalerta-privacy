1. Introducción

PagoAlerta es una aplicación móvil que permite a comercios capturar notificaciones de pagos recibidos a través de aplicaciones de pago peruanas (Yape, Plin, OH! Pay, BCP, BBVA, entre otras) y replicarlas en tiempo real a dispositivos secundarios dentro del mismo negocio.

Esta política explica qué datos maneja la aplicación, cómo los utiliza y qué derechos tienes como usuario. Nos tomamos en serio tu privacidad y la de tus clientes.

2. Datos que maneja la aplicación

2.1 Datos capturados de notificaciones

Cuando autorizas a PagoAlerta a leer notificaciones, la aplicación extrae únicamente los siguientes datos de las notificaciones de aplicaciones de pago:

  • Monto de la transacción (por ejemplo: S/ 25.00)
  • Nombre del remitente tal como aparece en la notificación (por ejemplo: "Juan Pérez")
  • Aplicación de origen (Yape, Plin, BCP, etc.)
  • Fecha y hora de la notificación

PagoAlerta no accede a las cuentas bancarias del usuario, ni a números de tarjeta, contraseñas, saldos, ni a ningún otro dato sensible más allá de lo que aparece en el texto de la notificación pública.

2.2 Datos de configuración

La aplicación almacena localmente en el dispositivo los siguientes datos de configuración:

  • Nombre del negocio que el usuario ingresa durante el setup
  • Nombre del dispositivo (emisor o receptor)
  • Preferencias de alertas (sonido, voz, vibración)
  • PIN numérico para proteger la configuración del webhook

2.3 Identificadores técnicos

  • ID anónimo de Firebase Authentication, generado automáticamente sin vincular a ninguna cuenta personal del usuario.
  • Token de Firebase Cloud Messaging (FCM) para envío de notificaciones push entre dispositivos del mismo negocio.

3. Cómo se usan los datos

Los datos recolectados se utilizan exclusivamente para:

  • Mostrar alertas de pago en los dispositivos receptores del mismo negocio.
  • Generar estadísticas y reportes diarios/por sesión de caja para el dueño del negocio.
  • Sincronizar pagos entre el dispositivo emisor y los dispositivos receptores del mismo negocio a través de Firebase Realtime Database.
  • Enviar notificaciones de pagos a un sistema externo del usuario (webhook configurado opcionalmente por el propio usuario).

La aplicación no utiliza los datos para publicidad, análisis de marketing, perfilamiento de usuarios, ni los vende a terceros.

4. Almacenamiento y seguridad

Los datos se almacenan en dos ubicaciones:

  • Localmente en el dispositivo: base de datos SQLite cifrada y SharedPreferences. Nunca salen del dispositivo a menos que sean explícitamente sincronizados con otros dispositivos del mismo negocio.
  • Firebase Realtime Database (Google): los pagos capturados se replican a Firebase exclusivamente para sincronizar los dispositivos receptores del mismo negocio. Toda comunicación con Firebase utiliza encriptación TLS/HTTPS.

El PIN de protección del webhook se almacena en el almacenamiento seguro del sistema operativo (Android Keystore) y nunca se transmite en texto plano.

5. Compartir datos con terceros

PagoAlerta no comparte datos con terceros para fines comerciales. Los únicos servicios externos con los que la aplicación se comunica son:

  • Google Firebase (proveedor de backend en la nube). Google procesa los datos según sus propios términos: firebase.google.com/support/privacy
  • Sistema externo del usuario, si y solo si el usuario configura manualmente un webhook en la sección "Conectar sistema". En ese caso, los datos se envían al endpoint que el propio usuario especifica, bajo su exclusiva responsabilidad.

6. Retención de datos

  • Los pagos en la base de datos local se eliminan automáticamente a los 30 días.
  • Los reportes diarios históricos se conservan hasta 1 año.
  • Los datos en Firebase se eliminan cuando el usuario ejecuta la opción "Desconfigurar" dentro de la aplicación.

7. Derechos del usuario

Como usuario tienes derecho a:

  • Eliminar todos tus datos utilizando el botón "Desconfigurar" en el menú de la aplicación. Esto borra los datos locales y los datos asociados en Firebase.
  • Revocar el permiso de acceso a notificaciones en cualquier momento desde los ajustes del sistema Android.
  • Solicitar información sobre los datos que la aplicación maneja, escribiendo a [hotdlolxp12@mail.com].

8. Menores de edad

PagoAlerta es una aplicación orientada a comercios y negocios. No está dirigida a menores de 13 años y no recopila conscientemente datos de menores de edad.
